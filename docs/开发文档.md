# 工作区模板 - 开发文档

本文档面向新接入的开发者，说明项目结构、约定与扩展方式，便于快速上手与协作。

---

## 一、项目概述

本工程是一个 **Vue 3 后台管理模板**，具备登录鉴权、布局与菜单、用户/角色/菜单管理（Mock）等基础能力，技术选型与交互参考 limsUI，便于在此基础上扩展业务。

**主要能力：**

- 登录 / 退出（含二次确认）、登录超时 401 自动跳转登录页
- 默认布局：侧栏菜单 + 顶栏（标题、用户姓名、退出）
- 系统管理：菜单管理（树形 CRUD + 侧栏联动）、用户管理、角色管理（含菜单授权）
- 请求封装：自动携带 token、GET 防缓存（时间戳 + 随机数请求头）、统一响应解包
- 预封装工具：日期、树结构、类型判断、文件下载、消息/确认框、页面跳转等

| 类别     | 技术 |

## 三、快速开始

```bash
# 安装依赖
pnpm install

# 开发
pnpm run dev
# 默认 http://localhost:3100，未配置 VITE_API_URL 时登录走 Mock

# 构建
pnpm run build

# 预览构建结果
pnpm run preview

# 清理缓存（异常时使用）
pnpm run clean:cache
```

**首次使用：** 不配置接口时，登录页输入任意用户名/密码即可进入；系统管理下菜单、用户、角色数据为 Mock（localStorage），刷新不丢失。

## 五、核心流程说明

### 5.1 登录与鉴权

1. **登录页** `src/views/sys/login/index.vue` 提交用户名、密码，调用 `useUserStore().login(params)`。
2. **userStore.login** 调用 `loginApi`（Mock 时从 localStorage 用户列表校验），成功后写入 **token** 与 **userInfo**（realname 等）到 store 与 sessionStorage。
3. **路由守卫** `setupRouterGuard()`（在 `main.ts` 中于 `app.use(router)` 之后调用）：
   - 访问需登录页面且无 token → 跳转 `/login?redirect=当前完整路径`。
   - 已登录访问 `/login` → 跳转 `redirect` 或 `/workbench`。
   - 路由 meta **ignoreAuth: true** 表示不校验 token（仅登录页使用）。
4. **顶栏用户姓名** 来自 `useUserStore().getRealname`，数据源为登录后写入的 userInfo。

### 5.2 请求与 401

- **封装入口：** `src/utils/http/axios/index.ts`，对外使用 **defHttp.get / defHttp.post**。
- **请求拦截：** 除 `requestOptions.withToken === false` 外，自动在请求头加上 token（`X-Access-Token`、`Authorization`）；GET 请求会加 **params._t** 和请求头 **X-TIMESTAMP**（时间戳+随机数）防缓存。
- **响应：** 默认按 **Result** 解包（`code === 200 || 0` 时返回 `result`，否则 reject）。
- **401 处理：** 响应拦截中若 status === 401，会清除 token/userInfo、跳转登录页并带上当前路径为 `redirect`，实现“登录超时自动退出到登录页”。

### 5.3 菜单与布局

- **侧栏数据来源：** `useMenuStore().menuTree`，由 `menuStore.fetchMenuTree()` 从菜单接口/Mock（localStorage）拉取并转为树形。
- **菜单管理页** 对菜单做增删改查（Mock 写 localStorage），每次变更后调用 `menuStore.fetchMenuTree()`，侧栏即时更新。
- **布局** 在 `src/layouts/default/index.vue`，挂载时拉一次菜单树，侧栏根据 `menuTree` 渲染；点击菜单项做 `router.push(path)`。

### 5.4 角色与菜单授权

- **角色管理** 列表/增删改使用 Mock（localStorage）；**授权** 打开抽屉，树形展示全部菜单，勾选后保存到 `ROLE_MENU_KEY`（结构为 `{ [roleId]: menuId[] }`）。
- 当前模板侧栏不按角色过滤，仅做“授权数据维护”；若后续要按角色显隐菜单，可在 `menuStore.fetchMenuTree()` 或布局中根据当前角色 id 与 `getRolePermission(roleId)` 过滤。

## 六、路由与 meta 约定

- **路由表** 在 `src/router/index.ts` 中集中定义；Layout 下子路由对应侧栏菜单（工作台、系统管理下各页）。
- **meta 约定：**
  - `title`：页面标题（可用于面包屑、标签页等）。
  - `ignoreAuth: true`：不校验登录，仅登录页使用。
- **默认首页：** 根路径 `/` 重定向到 `BASE_HOME`（`/workbench`），见 `enums/pageEnum.ts`。

## 七、请求封装使用说明

### 7.1 调用方式

```ts
import { defHttp } from '@/utils/http';

// GET（自动带 token、_t、X-TIMESTAMP）
const list = await defHttp.get<{ list: any[] }>({ url: '/api/xxx', params: { pageNo: 1 } });

// POST（自动带 token）
const res = await defHttp.post<{ id: string }>({ url: '/api/xxx', data: { name: 'xx' } });

// 登录等不需要 token 的接口
const token = await defHttp.post<{ token: string }>({
  url: '/sys/login',
  data: { username, password },
  requestOptions: { withToken: false },
});
```

### 7.2 请求选项（RequestOptions）

| 字段 | 说明 | 默认 |
|------|------|------|
| withToken | 是否带 token | true |
| joinTime | GET 是否加 _t 与 X-TIMESTAMP | true |
| isTransformResponse | 是否按 Result 解包取 result | true |

类型与统一返回结构见 `src/types/axios.d.ts`（Result\<T\>：code、message、result）。

### 7.3 接口 baseURL

来自环境变量 **VITE_API_URL**（`.env.development` 等）。不配置时为空，请求为相对路径；配置后所有未写死 host 的请求会拼上该 baseURL。

## 八、状态管理（Pinia）

### 8.1 用户 store：`useUserStore()`

- **state：** token、userInfo（username、realname 等，与 sessionStorage 同步）。
- **getters：** getToken、getRealname。
- **actions：** setToken、setUserInfo、login、logout。  
- 登出仅清空 token 与 userInfo，不负责跳转；跳转由布局“退出”或 axios 401 中执行。

### 8.2 菜单 store：`useMenuStore()`

- **state：** menuTree（侧栏用树形数据）。
- **actions：** fetchMenuTree()，从菜单 Mock/接口拉取并赋给 menuTree。  
- 布局与菜单管理页在适当时机调用 fetchMenuTree() 以更新侧栏。

## 九、工具与 Hooks

### 9.1 工具（utils）

- **is.ts：** isString、isObject、isArray、isEmpty、isNullOrUnDef 等，用于类型判断。
- **dateUtil.ts：** formatToDate、formatToDateTime（基于 dayjs），并导出 dayjs。
- **treeHelper.ts：** listToTree、treeToList，可配置 id、pid（默认 parentId）、children 字段名。
- **file/download.ts：** downloadByData（Blob 触发下载）、downloadByUrl（链接下载/新窗口）。
- 统一从 `@/utils` 引入（见 `utils/index.ts`），auth 单独从 `@/utils/auth` 引入。

### 9.2 Hooks

- **useMessage()：** createMessage（轻提示）、notification、createConfirm（确认框，可配 iconType 等）。
- **useGo()：** 页面跳转，`go(path)` 或 `go(path, true)` 表示 replace。  
- 从 `@/hooks` 引入。

### 9.3 @vueuse/core

按需从 `@vueuse/core` 引入，如 useClipboard、useStorage、useDebounceFn 等，不在此列举。

## 十一、新增页面/接口的推荐步骤

### 新增业务页面（需登录）

1. 在 `src/views/` 下新增页面组件（如 `views/biz/order/index.vue`）。
2. 在 `src/router/index.ts` 的 Layout.children 中增加路由：path、name、component、meta.title。
3. 若要在侧栏显示，需在 **菜单管理** 中新增对应菜单项（path 与路由 path 一致），或后续接后端菜单接口后由后端配置。

### 新增不需要登录的页面

1. 在 `src/views/` 下新增组件。
2. 在 router 中增加路由，并设置 **meta.ignoreAuth: true**。
3. 若希望不放在 Layout 内，可像登录页一样与 Layout 平级配置路由。

### 新增接口

1. 在 `src/api/` 下按模块建文件或追加方法，使用 `defHttp.get/post`，需登录的接口无需传参即自动带 token。
2. 登录、验证码等不需 token 的接口传 **requestOptions: { withToken: false }**。
3. 若后端返回格式与当前 Result 不一致，可对该接口传 **requestOptions: { isTransformResponse: false }** 自行处理 res.data。

- **开发：** `.env.development` 中可配置 `VITE_API_URL`（接口 baseURL）。未配置时登录等 Mock 使用 localStorage。

## 十三、常见问题

1. **登录后刷新姓名没了？**  
   userInfo 与 token 一样存在 sessionStorage，刷新会从 storage 恢复；若没有，检查登录成功后是否调用了 `setUserInfo(data.userInfo)` 以及 storage 是否被清空。

2. **请求 401 没跳转登录？**  
   检查 `src/utils/http/axios/index.ts` 响应拦截中 401 分支是否执行了 logout 与 router.push。

3. **侧栏菜单不更新？**  
   菜单管理或其它修改菜单的地方在保存/删除后需调用 `useMenuStore().fetchMenuTree()`。

4. **如何对接真实后端？**  
   配置 `VITE_API_URL`；将 `api/sys` 下各方法中的 Mock（localStorage）改为 `defHttp.get/post` 请求真实 URL；登录接口返回格式需包含 token（及可选 userInfo），与当前 `LoginResult` 一致或做一层适配。

5. **路径别名 @ 报错？**  
   确认 `tsconfig.json` 中 `compilerOptions.paths` 含 `"@/*": ["src/*"]`，且构建工具（Vite）中配置了相同别名（见 `vite.config.ts`）。
